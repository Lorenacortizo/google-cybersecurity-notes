# 📘 Module 4: Key Terms & Definitions

This module introduces fundamental concepts and tools commonly used in cybersecurity, especially in defensive roles such as SOC analysis.

---

## 🛡️ Security Tools & Systems

- **Antivirus Software**  
  Software used to prevent, detect, and remove malware and viruses.

- **Intrusion Detection System (IDS)**  
  An application that monitors system activity and alerts on potential intrusions.

- **SIEM (Security Information and Event Management)**  
  A system that collects and analyzes log data to monitor critical activities within an organization.

- **Network Protocol Analyzer**  
  A tool used to capture and analyze network traffic.

---

## 💾 Data & Information

- **Database**  
  An organized collection of data or information.

- **Data Point**  
  A single piece of specific information.

- **Log**  
  A record of events that occur within an organization’s systems.

---

## 🖥️ Systems & Technology

- **Linux**  
  An open-source operating system widely used in cybersecurity.

- **Programming**  
  The process of creating instructions for a computer to perform tasks.

- **SQL (Structured Query Language)**  
  A language used to create, manage, and query databases.

---

## 🔍 Digital Forensics & Incident Response

- **Order of Volatility**  
  A sequence that defines the order in which data should be preserved, from most to least volatile.

- **Evidence Preservation**  
  The process of properly handling and protecting fragile digital evidence.

---

## 🎯 Key Takeaways

- Logs and SIEM systems are essential for monitoring and detecting threats.  
- Tools like IDS and protocol analyzers help identify suspicious activity.  
- Understanding data (databases, SQL, logs) is critical for analysis.  
- Proper evidence handling is crucial during incident response.
